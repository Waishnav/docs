[GitCMS](https://gitcms.blog) est une extension Chrome qui transforme GitHub en un puissant CMS sans tête, offrant une expérience d'édition similaire à Notion pour gérer les sites statiques construits avec des frameworks comme Astro, Next.js, Hugo, Jekyll et bien d'autres. Il permet aux développeurs, clients et rédacteurs de contenu de gérer directement le contenu dans l'interface de GitHub sans serveurs supplémentaires ni configurations CMS complexes.

![GitCMS éditeur de type Notion](https://gitcms.blog/images/gitcms-editor-for-astro.webp)

## Prérequis
- Un projet Astro existant avec un dépôt GitHub.
- [Extension Chrome GitCMS](https://gitcms.blog) installée.

## Configuration de GitCMS
1. **Installer l'extension Chrome GitCMS** : Ajoutez l'[extension GitCMS](https://gitcms.blog) à Chrome.
2. **Initialiser GitCMS dans votre dépôt** :
   - Ouvrez votre dépôt GitHub.
   - Cliquez sur l'icône GitCMS dans le menu de l'extension Chrome.
   - Cliquez sur "Initialize".
   - Vous serez redirigé vers une nouvelle page de création de fichier GitHub avec `gitcms.md`, où l'éditeur de configuration GitCMS apparaîtra.

## Configuration de GitCMS
1. **URL du site web** : Entrez l'URL de votre site (ex: `https://yourblog.com`).
2. **Dossier des médias** : Spécifiez l'emplacement des fichiers médias (géré par votre générateur de site statique).

### Ajouter des collections
Les collections regroupent du contenu similaire, comme les articles de blog, pages ou membres d'équipe.

1. **Créer une collection** :
   - Cliquez sur "Add Collection".
   - Nommez la collection (ex: `posts`, `pages`, `authors`).
   - Spécifiez le dossier où les fichiers markdown sont stockés (ex: `src/content/posts`).

2. **Définir le schéma Frontmatter** :
   Configurez les champs de métadonnées :
   - **Champ de titre** : Pour les titres des articles.
   - **Champ à une ligne** : Pour du texte court comme les descriptions.
   - **Champ multi-lignes** : Pour du contenu plus long.
   - **DateTime** : Pour les dates de publication.
   - **MultiSelect** : Pour les catégories et balises.
   - **Champ de fichier** : Pour les téléchargements de médias.
   - **Champ booléen** : Pour activer/désactiver des options comme "brouillon".

   Exemple de configuration :
   ```yaml
   ---
   media_path: /public
   repo: YourUsername/your-repo
   website_url: https://yourwebsite.com
   
   collections:
     - name: posts
       file_extension: .md
       filename_pattern: title
       git_path: /src/content/posts
       metadata_schema:
         - name: title
           required: true
           type: title
         - name: description
           required: false
           type: multiline
         - name: date
           required: true
           type: datetime
         - name: image
           required: false
           type: file
         - allowed_values:
             - Product Updates
             - Features
             - Comparison
           name: categories
           required: true
           type: multi_select
         - allowed_values:
             - AuthorName
           name: authors
           required: true
           type: multi_select
         - name: draft
           required: false
           type: boolean
   ---
   ```
   GitCMS génère cela automatiquement, donc aucune édition manuelle n'est nécessaire.

## Utilisation de GitCMS
1. **Ouvrir l'éditeur GitCMS** :
   - Naviguez vers un fichier markdown dans votre dépôt.
   - Cliquez sur l'icône GitCMS pour basculer entre l'éditeur GitHub et GitCMS.

2. **Fonctionnalités d'édition** :
   - Édition markdown visuelle.
   - Formulaires de métadonnées structurés.
   - Téléchargement et prévisualisation de médias.

## Affichage du contenu avec Astro
Utilisez l'API Content Collections d'Astro pour afficher le contenu.

### Afficher une liste de collections
```tsx
---
import { getCollection } from 'astro:content';
const posts = await getCollection('posts');
---
<ul>
  {posts.map(post => (
    <li>
      <a href={`/posts/{post.slug}`}>{post.data.title}</a>
    </li>
  ))}
</ul>
```

### Afficher une seule entrée
```tsx
---
import { getEntry } from 'astro:content';

const post = await getEntry('posts', 'my-first-post');
const { Content } = await post.render();
---
<main>
  <h1>{post.data.title}</h1>
  <Content />
</main>
```
Pour plus d'informations sur la requête de contenu, consultez la [documentation sur les collections de contenu Astro](https://docs.astro.build/en/guides/content-collections/).

## Déploiement avec Astro
Référez-vous aux [guides de déploiement d'Astro](https://docs.astro.build/en/guides/deploy/) pour les options d'hébergement. Comme GitCMS stocke le contenu sur GitHub, le déploiement reste simple.
