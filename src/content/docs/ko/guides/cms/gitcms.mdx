[GitCMS](https://gitcms.blog)는 GitHub을 강력한 헤드리스 CMS로 변환하는 Chrome 확장 프로그램으로, Astro, Next.js, Hugo, Jekyll 등의 프레임워크로 구축된 정적 사이트를 관리할 수 있는 Notion과 유사한 편집 환경을 제공합니다. 추가 서버나 복잡한 CMS 설정 없이 개발자, 고객 및 콘텐츠 작성자가 GitHub 인터페이스에서 직접 콘텐츠를 관리할 수 있습니다.

![GitCMS Notion과 유사한 편집기](https://gitcms.blog/images/gitcms-editor-for-astro.webp)

## 사전 요구 사항
- GitHub 저장소가 있는 기존 Astro 프로젝트
- [GitCMS Chrome 확장 프로그램](https://gitcms.blog) 설치

## GitCMS 설정
1. **GitCMS Chrome 확장 프로그램 설치**: [GitCMS 확장 프로그램](https://gitcms.blog)을 Chrome에 추가하세요.
2. **GitCMS를 저장소에서 초기화**:
   - GitHub 저장소를 엽니다.
   - Chrome 확장 프로그램 메뉴에서 GitCMS 아이콘을 클릭합니다.
   - "초기화"를 클릭합니다.
   - `gitcms.md`라는 새로운 GitHub 파일 생성 페이지로 리디렉션되며, GitCMS 설정 편집기가 표시됩니다.

## GitCMS 구성
1. **웹사이트 URL**: 사이트의 URL을 입력하세요 (예: `https://yourblog.com`).
2. **미디어 폴더**: 정적 사이트 생성기가 미디어 파일을 저장하는 위치를 지정하세요.

### 컬렉션 추가
컬렉션은 블로그 게시물, 페이지 또는 팀 멤버와 같은 관련 콘텐츠 그룹을 나타냅니다.

1. **컬렉션 생성**:
   - "컬렉션 추가"를 클릭합니다.
   - 컬렉션 이름을 지정합니다 (예: `posts`, `pages`, `authors`).
   - 마크다운 파일이 저장될 폴더를 지정합니다 (예: `src/content/posts`).

2. **Frontmatter 스키마 정의**:
   메타데이터 필드를 구성합니다.
   - **제목 필드**: 게시물 제목
   - **한 줄 입력 필드**: 설명과 같은 짧은 텍스트
   - **여러 줄 입력 필드**: 긴 콘텐츠
   - **날짜 및 시간 필드**: 게시 날짜
   - **다중 선택 필드**: 카테고리 및 태그
   - **파일 입력 필드**: 미디어 업로드
   - **불리언 필드**: "초안"과 같은 토글

   예제 설정:
   ```yaml
   ---
   media_path: /public
   repo: YourUsername/your-repo
   website_url: https://yourwebsite.com
   
   collections:
     - name: posts
       file_extension: .md
       filename_pattern: title
       git_path: /src/content/posts
       metadata_schema:
         - name: title
           required: true
           type: title
         - name: description
           required: false
           type: multiline
         - name: date
           required: true
           type: datetime
         - name: image
           required: false
           type: file
         - allowed_values:
             - 제품 업데이트
             - 기능
             - 비교
           name: categories
           required: true
           type: multi_select
         - allowed_values:
             - 작성자명
           name: authors
           required: true
           type: multi_select
         - name: draft
           required: false
           type: boolean
   ---
   ```
   GitCMS가 자동으로 생성하므로 수동 편집이 필요하지 않습니다.

## GitCMS 사용하기
1. **GitCMS 편집기 열기**:
   - 저장소의 마크다운 파일로 이동합니다.
   - GitCMS 아이콘을 클릭하여 GitHub 및 GitCMS 편집기 간 전환합니다.

2. **편집 기능**:
   - 시각적 마크다운 편집
   - 구조화된 메타데이터 양식
   - 미디어 업로드 및 미리 보기

## Astro에서 콘텐츠 렌더링
Astro의 Content Collections API를 사용하여 콘텐츠를 표시합니다.

### 컬렉션 목록 표시
```tsx
---
import { getCollection } from 'astro:content';
const posts = await getCollection('posts');
---
<ul>
  {posts.map(post => (
    <li>
      <a href={`/posts/{post.slug}`}>{post.data.title}</a>
    </li>
  ))}
</ul>
```

### 단일 항목 표시
```tsx
---
import { getEntry } from 'astro:content';

const post = await getEntry('posts', 'my-first-post');
const { Content } = await post.render();
---
<main>
  <h1>{post.data.title}</h1>
  <Content />
</main>
```
자세한 콘텐츠 쿼리 방법은 Astro의 [Content Collections 문서](https://docs.astro.build/en/guides/content-collections/)를 참조하세요.

## Astro로 배포하기
GitCMS는 콘텐츠를 GitHub에 저장하므로 배포가 간단합니다. Astro의 [배포 가이드](https://docs.astro.build/en/guides/deploy/)를 참조하여 호스팅 옵션을 확인하세요.
