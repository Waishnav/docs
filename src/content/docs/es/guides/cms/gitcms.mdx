[GitCMS](https://gitcms.blog) es una extensión de Chrome que transforma GitHub en un potente CMS sin cabeza, proporcionando una experiencia de edición similar a Notion para gestionar sitios estáticos creados con frameworks como Astro, Next.js, Hugo, Jekyll y más. Permite a desarrolladores, clientes y redactores gestionar contenido directamente dentro de la interfaz de GitHub sin servidores adicionales ni configuraciones CMS complejas.

![Editor de GitCMS similar a Notion](https://gitcms.blog/images/gitcms-editor-for-astro.webp)

## Requisitos previos
- Un proyecto Astro existente con un repositorio de GitHub.
- [Extensión de Chrome GitCMS](https://gitcms.blog) instalada.

## Configuración de GitCMS
1. **Instalar la extensión de Chrome GitCMS**: Agrega la [extensión GitCMS](https://gitcms.blog) a Chrome.
2. **Inicializar GitCMS en tu repositorio**:
   - Abre tu repositorio de GitHub.
   - Haz clic en el icono de GitCMS en el menú de extensiones de Chrome.
   - Haz clic en "Inicializar".
   - Esto te redirigirá a una nueva página de creación de archivos en GitHub con `gitcms.md`, donde aparecerá el editor de configuración de GitCMS.

## Configuración de GitCMS
1. **URL del sitio web**: Ingresa la URL de tu sitio (ej. `https://tublog.com`).
2. **Carpeta de medios**: Especifica dónde se almacenan los archivos multimedia (generalmente gestionado por tu generador de sitios estáticos).

### Agregar Colecciones
Las colecciones agrupan contenido relacionado, como publicaciones de blog, páginas o miembros del equipo.

1. **Crear una Colección**:
   - Haz clic en "Agregar Colección".
   - Nombra la colección (ej. `posts`, `pages`, `authors`).
   - Especifica la carpeta donde se almacenan los archivos markdown (ej. `src/content/posts`).

2. **Definir el esquema de Frontmatter**:
   Configura los campos de metadatos usando:
   - **Campo de Título**: Para los títulos de publicaciones.
   - **Campo de Línea Única**: Para textos cortos como descripciones.
   - **Campo de Varias Líneas**: Para contenido extenso.
   - **Fecha y Hora**: Para fechas de publicación.
   - **Selección Múltiple**: Para categorías y etiquetas.
   - **Entrada de Archivo**: Para subir medios.
   - **Campo Booleano**: Para opciones como "borrador".

   Ejemplo de configuración:
   ```yaml
   ---
   media_path: /public
   repo: TuUsuario/tu-repositorio
   website_url: https://tusitio.com
   
   collections:
     - name: posts
       file_extension: .md
       filename_pattern: title
       git_path: /src/content/posts
       metadata_schema:
         - name: title
           required: true
           type: title
         - name: description
           required: false
           type: multiline
         - name: date
           required: true
           type: datetime
         - name: image
           required: false
           type: file
         - allowed_values:
             - Actualizaciones del Producto
             - Características
             - Comparación
           name: categories
           required: true
           type: multi_select
         - allowed_values:
             - Nombre del Autor
           name: authors
           required: true
           type: multi_select
         - name: draft
           required: false
           type: boolean
   ---
   ```
   GitCMS genera esto automáticamente, por lo que no es necesario editarlo manualmente.

## Usando GitCMS
1. **Abrir el Editor de GitCMS**:
   - Navega a un archivo markdown en tu repositorio.
   - Haz clic en el icono de GitCMS para alternar entre el editor de GitHub y el editor de GitCMS.

2. **Funciones de edición**:
   - Edición visual de markdown.
   - Formularios estructurados de metadatos.
   - Subida de medios con vista previa.

## Renderizando Contenido en Astro
Usa la API de Colecciones de Contenido de Astro para mostrar el contenido.

### Mostrar una Lista de Colecciones
```tsx
---
import { getCollection } from 'astro:content';
const posts = await getCollection('posts');
---
<ul>
  {posts.map(post => (
    <li>
      <a href={`/posts/{post.slug}`}>{post.data.title}</a>
    </li>
  ))}
</ul>
```

### Mostrar una Entrada Individual
```tsx
---
import { getEntry } from 'astro:content';

const post = await getEntry('posts', 'mi-primer-post');
const { Content } = await post.render();
---
<main>
  <h1>{post.data.title}</h1>
  <Content />
</main>
```
Para más información sobre consulta de contenido, consulta la [documentación de Astro sobre Colecciones de Contenido](https://docs.astro.build/en/guides/content-collections/).

## Desplegando con Astro
Consulta las [guías de despliegue de Astro](https://docs.astro.build/en/guides/deploy/) para opciones de alojamiento. Dado que GitCMS almacena contenido en GitHub, el despliegue sigue siendo sencillo.
